VulnRap: Comprehensive Scoring Overhaul — Development Prompt
Context for the AI Agent
You are working on VulnRap (vulnrap.com), a tool that analyzes vulnerability reports submitted to PSIRT (Product Security Incident Response Team) programs to determine if they are AI-generated "slop" — low-quality, hallucinated, or template-based reports produced by LLMs to collect bug bounties. The site is built on Replit and exposes a REST API at /api/reports/check (read-only analysis) and /api/reports (analysis + storage).
The Problem We Must Solve
We ran a rigorous blind test: 14 known AI slop reports and 15 confirmed legitimate human-written reports from HackerOne disclosures, oss-security mailing list (Daniel Stenberg, Qualys, ISC), and SEC Consult Full Disclosure advisories. The current heuristic scoring achieved a Cohen's d of 0.06 (negligible effect size) — essentially indistinguishable from random chance. The mean slop score was 32.6 and the mean legit score was 32.1, a gap of 0.5 points on a 0-100 scale. Best achievable accuracy at any threshold was 58.6%.
The root cause: the heuristics measure report formatting quality (missing versions, no code blocks, short length) rather than AI provenance signals. A polished fabricated report scores 18 while a terse real kernel developer report scores 48. The system trusts the fake and flags the real.
What the LLM scoring (llmSlopScore) should have caught but didn't
During testing, llmSlopScore was null on every single response and llmEnhanced was always false. This feature appears to be disabled or broken. Fixing this is the single highest-priority item because an LLM can detect patterns no heuristic can: hallucinated function names, fabricated stack traces, textbook recitations dressed as original analysis.

Scoring math for lexical markers:
pythondef score_lexical(text):
    text_lower = text.lower()
    total_weight = 0
    matches = []
    for phrase, weight in AI_PHRASES.items():
        if phrase in text_lower:
            total_weight += weight
            matches.append((phrase, weight))
    
    # Normalize to 0-100 with diminishing returns (log scale)
    # A single "certainly!" (10 pts) → ~25/100
    # Three medium markers (15 pts) → ~35/100
    # Five+ markers (30+ pts) → ~60-80/100
    import math
    normalized = min(100, 30 * math.log1p(total_weight))
    return normalized, matches
1B: Statistical Text Analysis
These features measure properties of the text that differ statistically between human and AI writing. The math matters here.
Sentence Length Variance
AI text has unusually uniform sentence lengths. Human text is bursty — short fragments mixed with long complex sentences.
pythonimport re
import numpy as np

def sentence_length_variance(text):
    """Lower variance = more likely AI. Returns a 0-100 suspicion score."""
    sentences = re.split(r'[.!?]+', text)
    sentences = [s.strip() for s in sentences if len(s.strip()) > 3]
    if len(sentences) < 5:
        return 0  # Not enough data
    
    lengths = [len(s.split()) for s in sentences]
    cv = np.std(lengths) / np.mean(lengths) if np.mean(lengths) > 0 else 0
    
    # Human writing typically has CV > 0.6
    # AI writing typically has CV 0.3-0.5
    # Map: CV < 0.3 → score 80, CV 0.3-0.5 → score 40-60, CV > 0.7 → score 10
    if cv < 0.3:
        return 80
    elif cv < 0.5:
        return 60 - (cv - 0.3) * 100  # linear interpolation
    elif cv < 0.7:
        return 30 - (cv - 0.5) * 50
    else:
        return max(5, 20 - (cv - 0.7) * 30)
Passive Voice Ratio
AI text uses more passive voice than humans, especially in technical writing.
pythondef passive_voice_ratio(text):
    """Higher passive ratio = more likely AI. Returns 0-100 suspicion score."""
    sentences = re.split(r'[.!?]+', text)
    sentences = [s.strip() for s in sentences if len(s.strip()) > 5]
    
    passive_patterns = [
        r'\b(?:is|are|was|were|been|being|be)\s+\w+ed\b',
        r'\b(?:is|are|was|were|been|being|be)\s+\w+en\b',
        r'\bhas been\b', r'\bhave been\b', r'\bwas found\b',
        r'\bwas discovered\b', r'\bwas identified\b',
        r'\bcan be exploited\b', r'\bcould be used\b',
    ]
    
    passive_count = 0
    for s in sentences:
        for pattern in passive_patterns:
            if re.search(pattern, s, re.IGNORECASE):
                passive_count += 1
                break
    
    ratio = passive_count / len(sentences) if sentences else 0
    
    # Human vuln reports: ~15-25% passive
    # AI vuln reports: ~35-55% passive
    if ratio > 0.5:
        return 75
    elif ratio > 0.35:
        return 50
    elif ratio > 0.25:
        return 25
    else:
        return 10
Contraction Absence Score
Real researchers use contractions ("doesn't", "can't", "won't"). AI defaults to formal language ("does not", "cannot").
pythondef contraction_absence_score(text):
    """Absence of contractions in informal context = AI signal."""
    contractions = ["don't", "doesn't", "can't", "won't", "isn't", "aren't",
                    "wasn't", "weren't", "couldn't", "shouldn't", "wouldn't",
                    "hasn't", "haven't", "hadn't", "it's", "that's", "there's",
                    "i'm", "i've", "i'd", "i'll", "we're", "we've", "we'd",
                    "they're", "they've", "you're", "you've"]
    
    formal_equivalents = ["do not", "does not", "cannot", "will not", "is not",
                         "are not", "was not", "were not", "could not",
                         "should not", "would not", "has not", "have not",
                         "had not", "it is", "that is", "there is",
                         "i am", "i have", "i would", "i will"]
    
    text_lower = text.lower()
    contraction_count = sum(1 for c in contractions if c in text_lower)
    formal_count = sum(1 for f in formal_equivalents if f in text_lower)
    
    total = contraction_count + formal_count
    if total < 3:
        return 0  # Not enough signal
    
    formality_ratio = formal_count / total
    # AI tends to have formality_ratio > 0.8
    # Humans tend to have formality_ratio < 0.5
    if formality_ratio > 0.9:
        return 60
    elif formality_ratio > 0.7:
        return 40
    elif formality_ratio > 0.5:
        return 20
    else:
        return 5
Perplexity Estimation (Optional but Powerful)
If you can run a small language model (GPT-2 tokenizer is sufficient), low perplexity = high AI likelihood. AI text is "predictable" to other models.
python# This requires: pip install transformers torch
# Only implement if you can run inference on the server

def perplexity_score(text):
    """Low perplexity = text is predictable = likely AI-generated."""
    from transformers import GPT2LMHeadModel, GPT2Tokenizer
    import torch
    
    tokenizer = GPT2Tokenizer.from_pretrained("gpt2")
    model = GPT2LMHeadModel.from_pretrained("gpt2")
    model.eval()
    
    tokens = tokenizer.encode(text[:1024], return_tensors="pt")
    with torch.no_grad():
        outputs = model(tokens, labels=tokens)
        loss = outputs.loss.item()
    
    perplexity = torch.exp(torch.tensor(loss)).item()
    
    # Human text: perplexity typically 50-200
    # AI text: perplexity typically 15-50
    if perplexity < 25:
        return 80
    elif perplexity < 40:
        return 55
    elif perplexity < 60:
        return 30
    else:
        return 10
If you cannot run a model on the server, use a simpler proxy: token entropy. Measure the Shannon entropy of word bigrams. AI text has lower entropy.
pythonfrom collections import Counter
import math

def bigram_entropy(text):
    """Lower entropy = more predictable = more likely AI."""
    words = text.lower().split()
    if len(words) < 20:
        return 0
    
    bigrams = [(words[i], words[i+1]) for i in range(len(words)-1)]
    counts = Counter(bigrams)
    total = len(bigrams)
    
    entropy = -sum((c/total) * math.log2(c/total) for c in counts.values())
    
    # Normalize by maximum possible entropy
    max_entropy = math.log2(total)
    normalized = entropy / max_entropy if max_entropy > 0 else 0
    
    # AI text: normalized entropy typically 0.85-0.92
    # Human text: normalized entropy typically 0.93-0.99
    if normalized < 0.88:
        return 65
    elif normalized < 0.92:
        return 40
    elif normalized < 0.95:
        return 20
    else:
        return 5

pythondef verify_cves(text):
    """Check if CVE IDs mentioned in the report actually exist."""
    cve_pattern = r'CVE-(\d{4})-(\d{4,})'
    cves = re.findall(cve_pattern, text)
    results = []
    
    for year, number in cves:
        cve_id = f"CVE-{year}-{number}"
        # Check against NVD API
        resp = requests.get(
            f"https://services.nvd.nist.gov/rest/json/cves/2.0",

            if data.get("totalResults", 0) == 0:
                results.append({
                    "type": "fabricated_cve",
                    "cve": cve_id,

                "type": "stale_cve",

            })
    return results

pythondef detect_severity_inflation(text):

3E: Fabricated Debug Output Detection
AI generates fake ASan/GDB/valgrind output. Real output has specific patterns that are hard to fake correctly.
pythondef detect_fake_debug_output(text):
    """Detect fabricated AddressSanitizer, GDB, or valgrind output."""
    signals = []
    
    # Check for ASan output
    asan_match = re.search(r'AddressSanitizer.*?0x([0-9a-f]+)', text)
    if asan_match:
        # Real ASan: addresses are in typical heap/stack ranges
        # Fake ASan: often uses round numbers or sequential addresses
        addr = asan_match.group(1)
        if addr.endswith('0000') or addr.endswith('000'):
            signals.append({"type": "suspicious_asan_address", "weight": 8})
    
    # Check for suspiciously clean stack traces
    # Real traces have library names, offsets; fake ones are too clean
    stack_frames = re.findall(r'#\d+\s+0x[0-9a-f]+\s+in\s+(\w+)', text)
    if len(stack_frames) >= 3:
        # Check if frames reference functions that all follow same naming pattern
        # and repeat (AI often generates repeating frames)
        if len(set(stack_frames)) < len(stack_frames) / 2:
            signals.append({"type": "repeating_stack_frames", "weight": 15})
    
    # Check for GDB output with fabricated register values
    gdb_regs = re.findall(r'(rax|rbx|rcx|rdx|rsi|rdi|rsp|rbp|rip)\s*=?\s*0x([0-9a-f]+)', text, re.IGNORECASE)
    if len(gdb_regs) >= 4:
        # Real registers are messy. Fake ones often have patterns.
        addrs = [g[1] for g in gdb_regs]
        if len(set(a[:4] for a in addrs)) <= 2:  # All addresses start the same
            signals.append({"type": "fabricated_registers", "weight": 10})
    
    return signals

Evaluate these dimensions:

2. ORIGINALITY (0-100): Does the analysis contain original insight, or does it read like a textbook/CWE definition repackaged as a "finding"? Real researchers describe what they actually did and discovered. AI regurgitates general knowledge about vulnerability classes.

AI often describes a vulnerability type correctly in general but makes errors connecting it to the specific target.

pythonimport json

async def llm_analyze(report_text, llm_client):
    """Run LLM analysis and return structured scores."""
    prompt = LLM_ANALYSIS_PROMPT.format(report_text=report_text[:4000])
    
    response = await llm_client.chat(
        model="gpt-4o-mini",  # or claude-3-haiku — fast and cheap
        messages=[{"role": "user", "content": prompt}],
        temperature=0.1,  # Low temp for consistent scoring
        max_tokens=500,
    )
    
    try:
        scores = json.loads(response.content)
        # Weighted combination of dimensions
        llm_score = (
            scores["specificity"] * 0.15 +
            scores["originality"] * 0.25 +
            scores["voice"] * 0.20 +
            scores["coherence"] * 0.15 +
            scores["hallucination"] * 0.25
        )
        return {
            "llmSlopScore": round(llm_score),
            "llmBreakdown": scores,
            "llmFeedback": scores.get("reasoning", ""),
            "llmRedFlags": scores.get("red_flags", []),
            "llmEnhanced": True,
        }
    except (json.JSONDecodeError, KeyError):
        return {"llmSlopScore": None, "llmEnhanced": False}

Score Fusion: Bayesian Combination
Combine all axes using calibrated weights. The key insight: different signals have different base rates and different reliabilities.
pythondef compute_final_score(linguistic, quality, factual, llm_score, 
                        linguistic_evidence, factual_evidence):
    """
    Combine all axes into a final calibrated slop probability.
    
    Uses a weighted approach with evidence-dependent confidence.
    """
    
    # Base weights (these should be tuned on labeled data)
    weights = {
        "linguistic": 0.25,
        "factual": 0.30,
        "llm": 0.35,
        "template": 0.10,
    }
    
    # Adjust weights based on evidence strength
    # If we found a hallucinated function name, that's near-conclusive
    has_hallucinated_ref = any(
        e["type"] == "hallucinated_function" for e in factual_evidence
    )
    if has_hallucinated_ref:
        weights["factual"] = 0.50
        weights["llm"] = 0.25
        weights["linguistic"] = 0.15
        weights["template"] = 0.10
    
    # If LLM scoring failed, redistribute its weight
    if llm_score is None:
        weights["linguistic"] += weights["llm"] * 0.4
        weights["factual"] += weights["llm"] * 0.4
        weights["template"] += weights["llm"] * 0.2
        weights["llm"] = 0
        llm_score = 0
    
    # Compute weighted score
    raw_score = (
        linguistic * weights["linguistic"] +
        factual * weights["factual"] +
        llm_score * weights["llm"] +
        # template score is part of linguistic but weighted separately
        linguistic * weights["template"]
    )
    
    # Confidence: how much evidence do we have?
    evidence_count = len(linguistic_evidence) + len(factual_evidence)
    confidence = min(1.0, 0.3 + evidence_count * 0.07)
    if llm_score is not None and llm_score > 0:
        confidence += 0.2
    confidence = min(1.0, confidence)
    
    # Apply confidence: uncertain scores get pulled toward the midpoint
    # This prevents overconfident scores when we have little evidence
    final_score = raw_score * confidence + 50 * (1 - confidence)
    
    return {
        "slopScore": round(final_score),
        "qualityScore": round(quality),  # SEPARATE — does not affect slop
        "confidence": round(confidence, 2),
    }

pythondef evaluate(predictions, ground_truth):
    """Compute classification metrics at various thresholds."""
    from sklearn.metrics import roc_auc_score, precision_recall_curve, f1_score
    
    # Binary labels: 1 = slop, 0 = legit
    y_true = [1 if gt == "slop" else 0 for gt in ground_truth]
    y_scores = predictions  # The slopScore values
    
    # AUC-ROC: the single best metric for discrimination ability
    # Current system: ~0.52 (near random). Target: >0.85
    auc = roc_auc_score(y_true, y_scores)
    
    # Cohen's d: effect size between slop and legit score distributions
    slop_scores = [s for s, gt in zip(y_scores, y_true) if gt == 1]
    legit_scores = [s for s, gt in zip(y_scores, y_true) if gt == 0]
    cohens_d = (np.mean(slop_scores) - np.mean(legit_scores)) / np.sqrt(
        (np.var(slop_scores) + np.var(legit_scores)) / 2
    )
    # Current: 0.06. Target: >1.0 (large effect)
    
    # Optimal F1 threshold
    precisions, recalls, thresholds = precision_recall_curve(y_true, y_scores)
    f1s = 2 * precisions * recalls / (precisions + recalls + 1e-8)
    best_threshold = thresholds[np.argmax(f1s)]
    best_f1 = np.max(f1s)
    # Current: ~0.55. Target: >0.80
    
    # False positive rate at 80% sensitivity (catch 80% of slop)
    # This is the metric PSIRT teams care about most:
    # "If I flag everything above threshold X, how much legit mail do I lose?"
    sorted_legit = sorted(legit_scores)
    threshold_80 = sorted([s for s, gt in zip(y_scores, y_true) if gt == 1])[
        int(0.2 * len(slop_scores))  # 80th percentile of slop = catches 80%
    ]
    fpr_at_80_sensitivity = sum(1 for s in legit_scores if s >= threshold_80) / len(legit_scores)
    # Current: ~73% FPR at 80% sensitivity (terrible). Target: <15%
    
    return {
        "auc_roc": round(auc, 3),
        "cohens_d": round(cohens_d, 2),
        "best_f1": round(best_f1, 3),
        "best_threshold": round(best_threshold, 1),
        "fpr_at_80_sensitivity": round(fpr_at_80_sensitivity, 3),
        "mean_slop": round(np.mean(slop_scores), 1),
        "mean_legit": round(np.mean(legit_scores), 1),
        "score_gap": round(np.mean(slop_scores) - np.mean(legit_scores), 1),
    }
